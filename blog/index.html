从零构建一个高性能 Web 服务器

一个用 C 语言编写的轻量级 HTTP 服务器，从最简单的单连接 TCP 模型出发，逐步演进为支持 Epoll I/O 多路复用、Nginx 风格 Master-Worker 多进程架构、HTTP/1.1 Keep-Alive 持久连接、完整静态网站托管与分离式日志系统的生产级服务器。

核心特性

六种并发模型

支持 TCP 单连接、fork-per-client、thread-per-client、线程池（动态扩缩 2→8）、Select、Epoll 共六种模式，可按需切换，便于性能对比与教学。

Master-Worker 架构

参考 Nginx 设计，Master 进程负责管理与信号分发，Worker 进程独立运行 Epoll 事件循环。fork 后 CoW 共享用户数据，多核并行处理。

HTTP/1.1 Keep-Alive

支持持久连接复用，兼容 HTTP/1.0 + Connection: Keep-Alive。SO_RCVTIMEO 空闲超时 + safe_close 优雅关闭，经过三轮 AB/wrk 调试。

静态网站托管

动态 Content-Type 映射（12 种 MIME 类型），多级目录，301/403/404/405 完整状态码处理。本 Blog 即运行在该服务器上。

分离式日志系统

系统日志与访问日志独立文件，四级日志（DEBUG/INFO/WARNING/ERROR），行数阈值自动滚动，printf 风格格式化 API。

多站点虚拟主机

参考 Nginx 的 server { } 块语法，基于 HTTP Host 头实现名称虚拟主机。一个 IP:端口托管多个独立网站，每个站点有独立的文档根目录和访问日志。

用户管理系统

支持 10 万+ 用户 CSV 数据集，链表 + 红黑树 BST 双索引，提供 find/add/delete/index/compare 完整 CRUD 操作。
