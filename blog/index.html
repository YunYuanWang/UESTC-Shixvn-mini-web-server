从零构建一个高性能 Web 服务器

本项目实现了一个支持 HTTP/1.1、Keep-Alive、Epoll I/O 多路复用、Master-Worker 进程模型的轻量级 Web 服务器。

核心特性

Epoll I/O 多路复用

基于 Linux epoll 实现高效的 I/O 事件驱动，单进程即可管理数千并发连接，相比 select/poll 具有 O(1) 的事件分发性能。

Master-Worker 架构

参考 Nginx 设计，Master 进程负责管理，Worker 进程处理请求。支持多核并行，优雅重启，信号驱动的进程管理。

HTTP/1.1 Keep-Alive

支持持久连接，多个请求复用同一 TCP 连接。经过 3 轮调试，兼容 ApacheBench 和 curl 的不同大小写约定。

多模式服务器

支持多种并发模型：epoll、select、fork-per-client、thread-per-client、线程池，便于性能对比和学习。
