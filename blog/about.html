整体架构

Mini Web Server 采用 Master-Worker 多进程模型，参考了 Nginx 的设计理念：

                    ┌─────────────┐
                    │   Master    │
                    │   Process   │
                    └──────┬──────┘
                           │ fork()
              ┌────────────┼────────────┐
              ▼            ▼            ▼
        ┌──────────┐ ┌──────────┐ ┌──────────┐
        │ Worker-1 │ │ Worker-2 │ │ Worker-N │
        │ (Epoll)  │ │ (Epoll)  │ │ (Epoll)  │
        └──────────┘ └──────────┘ └──────────┘
              │            │            │
              └────────────┼────────────┘
                           ▼
                    ┌─────────────┐
                    │  Listen FD  │
                    │  (SO_REUSEADDR) │
                    └─────────────┘

Epoll 事件驱动

每个 Worker 进程运行独立的 epoll 事件循环：

  • epoll_create1(0): 创建 epoll 实例
  • epoll_ctl(ADD): 注册 listen_fd 和 client_fd 到 epoll
  • epoll_wait(): 阻塞等待事件，O(1) 事件分发
  • Level-Triggered (LT): 默认水平触发模式，与 select 语义兼容

Keep-Alive 连接管理

每个连接维护状态：fd、IP、port、recv_buf、request_count、last_activity。超时空闲连接（5秒）自动清理，最大请求数限制（10,000次）防止资源泄漏。

优雅关闭

safe_close() 机制：shutdown(SHUT_RDWR) → drain → close()，避免 TCP RST 包。Master 发送 SIGTERM → Worker 优雅退出 → 超时后 SIGKILL 强制终止。
