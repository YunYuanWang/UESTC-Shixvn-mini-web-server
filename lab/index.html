深入 Linux 网络编程的实验室

这是一个用于学习和实验 Linux 系统编程的技术站点。从 Socket API 到 Epoll 事件驱动，从进程管理到 HTTP 协议实现——这里记录了构建一个高性能 Web 服务器的完整探索过程。

实验模块总览

🧵 并发模型实验

对比六种并发策略：单连接阻塞 → fork-per-client → thread-per-client → 线程池 → Select 多路复用 → Epoll 事件驱动。理解每种模型在不同并发量下的性能特征。

⚡ I/O 多路复用

从 select() 的 fd_set 位图扫描到 epoll 的 O(1) 事件交付。实验包括 LT/ET 触发模式对比、惊群效应测试、连接超时管理与 Keep-Alive 协商。

🏗️ 进程架构设计

参考 Nginx 的 Master-Worker 模式：fork() 共享监听 socket、SIGTERM 优雅关闭、SIGCHLD 僵尸回收、CoW 内存共享。理解多进程服务器的生命周期管理。

🌐 HTTP 协议栈

手写 HTTP/1.1 解析器：请求行 + 头部解析 + Keep-Alive 协商 + 动态 MIME 映射 + 301/304 缓存重定向 + 分块传输。从 TCP 字节流到完整 HTTP 语义。

📊 性能基准测试

使用 ApacheBench (ab)、wrk、wrk2 进行压测。QPS 吞吐量、P50/P99 延迟、连接复用率对比。Keep-Alive ON: 2,543 req/s vs OFF: 764 req/s（3.3× 提升）。

🔧 系统调优

SO_REUSEADDR 快速重启、TCP_NODELAY 延迟优化、SO_RCVTIMEO 超时配置、文件描述符限制调优、日志滚动策略。将理论应用到实际系统。

快速开始

# 构建项目
make clean && make

# 启动 Master-Worker 模式（推荐）
./mini_web_server master conf/server.conf

# 测试 API
curl http://127.0.0.1:80/hello
curl http://127.0.0.1:80/users

# 浏览器访问静态站点
# http://127.0.0.1:80/        → www/ 主页
# http://127.0.0.1:80/blog/   → blog/ 博客
# http://127.0.0.1:80/lab/    → lab/ 实验室（本页）

# 压力测试
ab -k -c 50 -n 10000 http://127.0.0.1:80/hello
